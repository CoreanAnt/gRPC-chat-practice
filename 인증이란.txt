클라이언트 → 서버: 익명으로 API 호출.

서버 → 클라이언트: "인증이 필요합니다(401 Unauthorized)" 응답과 함께 인증 서버(AS)의 주소를 알려줌.

클라이언트 → 인증 서버 (리디렉션): 사용자의 브라우저를 통해 인증 서버로 이동합니다. 이때 Client ID(클라이언트 앱 식별자) 등의 정보를 URL 파라미터로 보냅니다.

사용자 식별: 사용자가 인증 서버에 로그인합니다.

인증 서버 → 클라이언트: 인증 성공 후, 클라이언트에게 바로 토큰을 주는 것이 아니라 **인가 코드(Authorization Code)**를 전달하며 다시 클라이언트로 리디렉션시킵니다.

클라이언트 → 인증 서버: 클라이언트가 받은 '인가 코드'와 자신의 'Secret Key'를 가지고 인증 서버에 **액세스 토큰(Access Token)**을 요청합니다. (보안을 위한 2중 확인 단계)

클라이언트 → 서버: 획득한 토큰을 Authorization 헤더에 담아 API를 호출합니다.
